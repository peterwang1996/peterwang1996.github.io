一次用研笔试的作答：大学生兼职 APP 的用户招募条件 & 用户招募公告

发表于 2018-04-03

按：这是我前段时间投递某咨询公司时收到的一份测题，但最后因为一些意外的原因并没有能够去参与面试。但我觉得这个测题本身非常有趣，所以在征得对方同意后，将这份测题以及我的作答整理出来在这里分享。

本回答共分别为 用户招募条件确定、C端用户（大学生）招募公告、B端用户（雇主）招募公告、C端用户（大学生）招募问卷及设计说明、B端用户（雇主）招募问卷及设计说明 五个部分。其中两个招募问卷不仅有包含在本回答中、附有说明的版本，点击招募公告中的对应链接还可以访问对应招募问卷的在线版本。

题目：A公司拟开发一款大学生兼职平台APP（大学生可找各类兼职），打算邀请8-10为用户进行访谈。请思考需要的用户类型，制定用户招募条件，并制作成一份对外发布的《用户招募公告》。

用户招募条件确定

考虑到要开发的是一个平台类的产品，所以用户的大类可以分为C端用户和B端用户，再结合具体场景，不难发现C端用户实际上是希望通过平台找到兼职的大学生，B端用户是希望通过平台发布兼职信息的雇主。

通过对目前已有的大学生查找兼职信息 / 雇主发布兼职信息的手段进行分析和汇总，我们不难发现兼职信息的供求双方主要通过以下手段来交换兼职信息：

  1. 已在兼职的大学生获得了自己雇主的兼职信息，与同学朋友进行交流
  2. 雇主在大学生密集的地方（如学校宿舍区）通过张贴广告 / 派发传单来发布兼职信息
  3. 通过线下的专门场所（如人才市场）来发布 / 了解信息
  4. 雇主通过联系学生群体中的意见领袖（团体或个人，如社团 / 学生会 / 班干部 / 老师）或雇用校园代理来扩散兼职信息
  5. 雇主在大学生常用的线上社区（如学校贴吧 / 论坛 / 兼职QQ群）或行业线上社区（如互联网行业社区V2EX）来发布兼职信息
  6. 通过“智联招聘”等传统互联网招聘平台（或企业自有的平台）进行兼职信息的发布 / 获取
  7. 通过“兼职猫”等专业的互联网兼职平台进行兼职信息的发布 / 获取

结合访谈需求对以上手段的梳理和综合，我们可以大致将潜在用户做如下的分类：

  1. 没有线上发布 / 获取兼职信息经验的
  2. 有线上发布 / 获取兼职信息经验，但没有使用过线上招聘平台的
  3. 使用过传统线上招聘平台，但没有使用过专业互联网兼职平台的
  4. 使用过专业互联网兼职招聘平台的

对于作为C端用户的在校大学生，可以想到，对于以上四个类别，第一种是数量较少，且相对较难转化为目标产品的用户的，结合访谈人数的限制，在这里不做考虑，而后三种有在线上寻找兼职信息的用户都有成为目标产品用户的潜质，所以将其纳入期待的访谈对象。此外，从未有过兼职经历的大学生和已经有过兼职经历（但是有兼职愿望）的大学生，在搜寻兼职信息上，无论是在搜寻渠道和习惯上，还是筛选信息的侧重点上都会有所不同。所以我们的访谈对象有必要同时覆盖到这两类用户。

对于作为B端的雇主，我们可以同样地分成两类，一类是像小餐馆这类的个体户，另一类是相对大的公司/企业的HR。对于这两类雇主，他们都有可能使用过专业的兼职招聘平台。但对于大公司的HR来说，相对使用在网络社区发布实习信息的方式，他们更有可能会通过传统的线上招聘平台来进行兼职信息发布，而对于个体户则恰好相反。并且这两类用户在招聘兼职大学生时关心的问题也会有所不同。在对访谈对象进行选择的时候，应该注意到这些问题。

综上所述，我认为分配这十个访谈对象的一种可行的方案为：

  1. C端用户（希望通过平台找到兼职的大学生，共6人）
    1. 有线上获取兼职信息经验，但没有使用过线上招聘平台，第一次找兼职的用户
    2. 有线上获取兼职信息经验，但没有使用过线上招聘平台，有过兼职经验的用户
    3. 使用过传统线上招聘平台，但没有使用过专业互联网兼职平台，第一次找兼职的用户
    4. 使用过传统线上招聘平台，但没有使用过专业互联网兼职平台，有过兼职经验的用户
    5. 使用过专业互联网兼职招聘平台的，第一次找兼职的用户
    6. 使用过专业互联网兼职招聘平台的，有过兼职经验的用户
  2. B端用户（希望通过平台发布兼职信息的雇主，共4人）
    1. 使用过专业互联网兼职招聘平台的大企业HR
    2. 使用过专业互联网兼职招聘平台的个体户
    3. 使用过传统线上招聘平台，但没使用过专业互联网兼职招聘平台的大企业HR
    4. 有线上发布兼职信息经验，但没使用过专业互联网兼职招聘平台的个体户

除此之外，满足以下这些要求的访谈对象能够帮助我们更加顺利地推进访谈过程：

  1. 目前正在发布 / 获取兼职信息，或者在两周之内有这样经历的
  2. 愿意且能够清晰地表达出自己想法的
  3. 居住地在公司附近的或愿意来公司参加访谈的（方便面访）
  4. 有空的参加访谈的时间与公司安排的访谈时间相匹配的

[用户招募条件部分结束]

C端用户（大学生）招募公告

A 公司用户访谈活动——邀请正在找兼职的你！

我们是谁

我们是 A 公司的研发团队，为了全面地了解目前大学生网上兼职信息供求情况，需要邀请若干名在校大学生，在下周参与一个大约持续一个小时的访谈。

通过参与这次访谈，你能得到……

  1. A 公司的一份精美的周边礼物或 100 元酬金
  2. 一次参观 A 公司的机会

我们希望你……

  1. 是在校大学生
  2. 目前正在网上找兼职工作，或者是过去两周内有过在网上找兼职工作的经历
  3. 下周（3月19日至3月25日之间）在成都，且有空参加访谈

如果你符合以上要求，并希望参与这次活动……

请点击这个链接填写一份简短的问卷。在你完成问卷后，我们的工作人员将会通过电话与你取得联系。

你的真实想法对我们非常重要，感谢你的参与！

[招募公告结束]

B端用户（雇主）招募公告

A 公司用户访谈活动招募兼职大学生雇主 / 招聘专员

我们是 A 公司的研发团队，为了全面地了解目前大学生网上兼职信息供求情况，需要邀请若干名雇主和企业的招聘专员，在下周参与一次访谈。访谈大约持续一个小时，完成访谈后您将获得100元酬金作为感谢。

我们正在寻找这样的用户：

  1. 您（或您的单位）正在网上发布消息招聘在校兼职大学生，或者在前两周有过这样的经历
  2. 下周（3月19日至3月25日之间）在成都，且有空参加访谈

如果您符合以上要求，并希望参加这次访谈，请点击这个链接填写一份简短的问卷。在您完成问卷后，我们的工作人员将会通过电话与您取得联系。

您的真实想法对我们非常重要，感谢您的参与！

[招募公告结束]

C端用户（大学生）招募问卷及设计说明

A 公司用户访谈招募问卷

您好！感谢您对这次访谈活动的关注！

如果您希望参加这次访谈活动，请您认真填写这份问卷，方便我们在与您联系之前，对您有一个大致的了解。

完成这份问卷大约需要5~10分钟，感谢您的付出！

您目前就读的年级是？[单选题]

  1. 大一
  2. 大二
  3. 大三
  4. 大四
  5. 研一
  6. 研二
  7. 研三
  8. 我不是在校本科/专科/硕士生

    设计此题的目的是为了筛除非在校大学生，即选择了第8项的人群。

以下哪个选项的描述符合您目前的情况？[单选题]

  1. 我没有做过兼职，但最近正在寻找兼职的机会
  2. 我做过兼职，而且最近正在寻找兼职的机会
  3. 我在最近的两个星期之内找到了第一份兼职
  4. 我做过不止一次兼职，而且最近的一次兼职是在最近两周找到的
  5. 以上描述均不符合我的情况

    此题的1、3两项对应第一次找兼职的人群，2、4两项对应已有兼职经验的人群，选择5的人群则应被筛除。

在您收集兼职信息时，哪些手段是您正在或曾经使用到的？[多选题]

  1. 同学间交流了解
  2. 看到街边的传单 / 张贴广告
  3. 出席线下的招聘场所（如招聘会或人才市场）
  4. 从社团 / 学生会 / 班干部 / 老师处了解
  5. 从专门的校园兼职代理处了解
  6. 通过学校贴吧 / 论坛 / 兼职QQ群或其他社交媒体（如微博 / 论坛）了解
  7. 通过专门的招聘网站 / APP了解
  8. +其他（请注明）____

    选择了此题第6项（未选第7项）的对应有线上获取兼职信息经验，但没有使用过线上招聘平台的人群，选第七项的对应使用过互联网招聘平台的人群，还需要在下一题做细分。未选6、7两项的人群应被筛除。此外，选择了“其他”的应根据其填入的内容酌情处理。

您在收集兼职信息时，曾使用过 / 正在使用的网站 / APP有哪些？[多选题]

  1. 前程无忧
  2. 智联招聘
  3. 58同城
  4. 大街网
  5. 51Job
  6. 兼职猫
  7. 微兼职
  8. 同学帮帮
  9. e兼职
  10. +其他____

    选择了此题第1~5项（未选择第6~9项）对应使用过传统线上招聘平台，但没有使用过专业互联网兼职平台的人群，选择了第6~9项对应使用过使用过专业互联网兼职平台的人群。此外，选择了“其他”的应根据其填入的内容酌情处理。

请用几句话告诉我们，您在找兼职的过程中遇到过的困难或麻烦有哪些。[多行文本题]

此题用于考察用户的思维的清晰性以及表达的意愿和流畅性。处于访谈成本的考虑，应优先考虑邀请在此题认真作答，并且答案清楚明白，且答案长度不少于两句话的用户进行访谈。

您就读的大学是？[联动题，选项略]

此题用于考察是否方便邀请此用户进行当面访谈。

请告诉我们您希望接受访谈的时间[单行文本题](3月19日至3月25日之间，输入示例：3月20日下午)
